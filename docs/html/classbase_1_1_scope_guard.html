Ausführliche Beschreibung

Hilfsklasse zur Sicherstellung, dass eine bestimmte Funktion spätestens bei der Zerstörung des Objektes ausgeführt wird.

Diese Objekte können beispielsweise dort angewendet werden, wo am Ende eines Vorgangs bestimmte Ressourcen wieder freigegeben werden müssen, ungeachtet des Ausstiegspunktes und vor allem auch im Fehlerfall. Wo es kein Objekt gibt, dessen Destruktor für die Abarbeitung genutzt werden kann, oder wenn der auszuführende Code für einen Destruktor ungeeignet ist, kann stattdessen ein ScopeGuard-Objekt verwendet werden.

void base::ScopeGuard::Clear ( )
inline

Entfernt die Release-Funktion. Wird nun Release() aufgerufen oder das Objekt zerstört, wird kein zusätzlicher Code mehr ausgeführt, wenn nicht zuvor eine andere Release-Funktion über die Methode Releaser() angegeben wurde.

void base::ScopeGuard::Release ( )
inline

Führt die Release-Funktion aus, sofern definiert.

Wenn nach dem Aufruf von Release() keine weitere Funktionszuweisung über Releaser() mehr erfolgt, wird bei der Zerstörung des Objektes kein zusätzlicher Code mehr ausgeführt. Gleiches gilt nach Verwendung der Methode Clear().
